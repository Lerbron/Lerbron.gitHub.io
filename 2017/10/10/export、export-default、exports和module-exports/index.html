export、export default、exports和module.exports

require: node 和 es6 都支持的引入
export / import : 只有es6 支持的导出引入
module.exports / exports: 只有 node 支持的导出

在一个node执行一个文件时，会给这个文件内生成一个 exports和module对象，而module又有一个exports属性。

exports = module.exports = {}。

然而，exports 只是 module.exports的引用，辅助后者添加内容用的。
1.export与export default均可用于导出常量、函数、文件、模块等
2.在一个文件或模块中，export、import可以有多个，export default仅有一个
3.通过export方式导出，在导入时要加{ }，export default则不需要
4.export能直接导出变量表达式，export default不行。

模块化的作用：
1、代码分离。代码按照不同的功能分离成不同的小块，易于被理解；
2、提高复用性。可以被不同的文件所引用，进行组合，灵活性增加；
3、解决全局变量污染问题

现有模块化方案：
es6、AMD、CMD、CommonJS
1、es6编译时就能确定依赖关系，es6模块遇到import语句，不是去执行模块，而会生成一个引用，等到用的时候，才会去模块中取值。因为是动态引用，所以不会有缓存。
2、CommonJs只能在运行时确定，同步加载，模块加载的顺序按照其在代码中出现的顺序进行
3、AMD异步加载模块，依赖前置，比较适合浏览器环境，实现 js 文件的异步加载，避免网页失去响应，管理模块之间的依赖性，便于代码的编写和维护，代表库: RequireJS
4、CMD依赖就近，需要用到依赖的时候才申明，代表库: Sea.js

ES6 模块与 CommonJS 模块的差异
CommonJS 模块输出的是一个值的拷贝，ES6 模块输出的是值的引用。
CommonJS 模块是运行时加载，ES6 模块是编译时输出接口。
ES6 模块是动态引用，并且不会缓存值。

ES6 模块在对脚本静态分析的时候，遇到 import 就会生成一个只读引用，等到脚本真正执行的时候，再根据这个只读引用，到被加载的那个模块里取值，所以说 ES6 模块是动态引用。
从依赖中引入的模块变量是一个地址引用，是只读的，可以为它新增属性，可是不能重新赋值。
